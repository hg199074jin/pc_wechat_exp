# 知识沉淀规则

## 数据模型（二维状态）

**review 维度**（`knowledge_cards.status`）：
- `inbox` 待审 / `saved` 已收藏 / `archived` 已归档 / `rejected` 已忽略

**lifecycle 维度**（`knowledge_cards.lifecycle_stage`）：
- `captured` 默认 / `ingested` 已吸收 / `transformed` 已转化 / `applied` 已应用

两个维度独立，不互相覆盖。旧数据迁移后全归 `captured`。

## 不可变原则

- **转化创建 derivative，不覆盖原卡**：原卡 `content_md`/`type` 永不因转化改变
- derivative kind 有限集合：`my_version`(个人诠释) + `audit_case/sop/prompt/faq/article/script`(结构化重写)
- card type 与 derivative kind **不可互换**

## agent_rules

- 与 card 分离：一卡可生多规则，编辑规则不改原知识
- 状态：`draft` → `published` → `archived`
- **LLM 只生成草案，不能发布**；发布必须人工确认
- `source_card_id` 用 `ON DELETE RESTRICT`（应用层先拦截409，DB兜底）
- `knowledge_derivatives.card_id` 用 `ON DELETE CASCADE`（derivative 随源卡消失）

## 删除

- 删带规则的卡：应用层检查返回结构化 409，不抛原始 SQLite 外键错误
- derivative 随卡级联删除
