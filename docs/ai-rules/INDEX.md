# AI Rules Index

本目录给 Codex / Cursor / 其他 AI 编程助手提供本项目稳定、短小的规则。

**重要**：`AGENTS.md` 是导航层，不是知识库本体。本目录存放规则，更大的知识沉淀在
`knowledge.db` 和 Obsidian vault 里。规则要短、稳定；频繁变化的内容不放这里。

## 规则文件

| 文件 | 适用场景 |
|------|---------|
| `01_项目定位.md` | 新功能设计、产品方向判断、需求取舍 |
| `02_代码结构.md` | 修改代码、新增组件、重构、排查构建错误 |
| `03_AI分析规则.md` | 修改 AI 群聊分析逻辑、artifact、证据校验 |
| `04_知识沉淀规则.md` | 修改知识雷达、卡片、derivative、agent_rules |
| `05_Obsidian同步规则.md` | 修改 Obsidian 导出、增量同步、双链 |
| `06_隐私合规规则.md` | 涉及数据上传、LLM 调用、敏感信息处理 |
| `07_中文编码规则.md` | 处理中文内容、生成文档、避免乱码 |
| `08_我的长期偏好.md` | 沟通风格、输出结构、知识转化偏好 |

## 如何使用

修改代码前，先判断本次任务属于哪类（UI/功能/Bug/文档/AI逻辑/数据处理），
然后阅读对应规则文件。涉及多类时全部阅读相关文件。
