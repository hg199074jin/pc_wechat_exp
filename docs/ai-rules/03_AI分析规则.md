# AI 分析规则

## artifact 流程

`ai_analyzer.py` 的 `analyze_group` / `analyze_group_range`：
取消息 → 格式化(带msg_id) → 风格指纹注入 → (分块→rollup) → LLM产出JSON artifact → 证据校验 → 存盘 → 渲染Markdown

## 关键约束

- **证据校验是软门**：报告 msg_id/sender/quote 与源消息比对，未通过只标记不删除
- **风格指纹只存群级**：群定位/主题/黑话/偏好/禁忌，**不存个人画像**
- **超长消息分块**：>12000字符按行切分(每块~6000，最多8块)，分块产出后 rollup 合并去重
- **JSON 自愈**：正则预处理(去尾随逗号/注释) → 严格解析 → 失败让 LLM 修复最多2次
- **date 参数必须净化**：`_safe_date` 挡住路径穿越，只允许数字/连字符/字母/_to_

## LLM 调用

- `call_llm` 支持 `proxy`(auto/none/自定义)，国内服务用 none 绕过系统代理
- ProxyError/ConnectionError/5xx 重试3次+退避
- 测试连接/分析/分类/修复都从 cfg 透传 proxy

## 不要做

- 不让 LLM 直接产出 `.md` 报告（必须先产 artifact JSON 再渲染）
- 不在 artifact 里编造 msg_id（必须来自输入 `[msg_id=数字]`）
- 不存个人长期标签
