# 代码结构

## 分层

- **引擎层** `src/engine/services/`：纯逻辑，被 Web 和 CLI 共享，**不依赖 Flask**
- **Web 层** `src/web/`：Flask 路由保持薄，持久化/导出逻辑放引擎层
- **前端** `src/web/static/js/`：vanilla JS，按页面拆分

## 约定

- Python 导入用扁平风格：`from engine.services.xxx import yyy`
- 所有 API 路由返回 JSON，用 Flask Blueprint
- 引擎层函数签名清晰、可独立测试
- 前端动态内容必须转义后再插入 DOM，禁止 innerHTML 插入原始 LLM JSON

## SQLite 使用

- knowledge.db 用 WAL 模式 + `PRAGMA busy_timeout=5000`
- 连接必须 `try/finally` 关闭
- `init_db` 缓存是 schema-versioned 的，老库迁移不被跳过
- 写文件用原子写（`.tmp` + `os.replace`）

## 不要做

- 不在引擎层 import flask
- 不引入不必要的新依赖
- 不删除已有功能除非明确要求
- 不把简单问题复杂化
