# 项目定位

WeChat EXP 不只是聊天备份工具。核心价值是把高价值群聊讨论转化为**可溯源、可复用、AI 可读**的知识，且不允许未审 LLM 输出直接成为长期开发规则。

## 核心闭环

```
聊天记录 → AI 分析 → 知识卡片(审阅) → 我的版本(转化) → 规则草案(人工发布) → AI 可读规则文件
```

## 关键边界

- `knowledge.db` 是卡片/derivative/规则的**唯一真源**
- Obsidian/llm-wiki 是**生成的、增量的、人可读的视图**，不是真源
- 原始群内容和未审 LLM 输出**绝不**直接写入任何项目的 `AGENTS.md`
- 发布规则 = 导出到共享 vault；**不自动修改**其他仓库的 `AGENTS.md`

## 取舍原则

- 优先服务长期目标：审计能力、AI 工具化、个人知识库
- MVP 优先，避免过度设计
- 不破坏现有备份/解密/查看主流程
