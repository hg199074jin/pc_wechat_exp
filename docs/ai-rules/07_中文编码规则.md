# 中文编码规则

本项目所有文本文件统一使用 **UTF-8** 编码。

## 必须遵守

1. 不把 UTF-8 文件改写成 GBK、ANSI 或其他编码
2. 新建 `.md`/`.txt`/`.json`/`.py`/`.js`/`.html` 文件一律 UTF-8
3. Python 读写文件显式指定 `encoding='utf-8'`
4. 写 JSON 用 `ensure_ascii=False`（保留中文可读）
5. 发现中文乱码，先判断源文件编码，**不要直接覆盖原文件**
6. 修改中文文档前，先读取确认内容显示正常

## Windows 特定

- cmd 控制台默认 GBK，输出中文可能乱码（程序内部已处理）
- 路径含中文/`@chatroom` 时用 `_safe_id` 净化为文件系统安全名
- `os.path.join` 在 Windows 用 `\`，注意路径穿越防护（`_safe_date`）

## 不要做

- 不用 `encoding='gbk'` 或省略 encoding 参数
- 不在 git 提交信息里混入非 UTF-8 字符
- 不假设终端能正确显示所有 Unicode（emoji/罕见字可能显示异常）
